This is a Movie API App.

To build the server-side component of a “movies” web application. The web application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

Whenever users of mycommedyFlix are interacting with the application, the server-side of the application will be in use, processing their requests and performing operations against the data in the database. Users will be able to use the mycomedyFlix application whenever they like to read information about different movies or update their user information, for instance, their list of “Favorite Movies.”

Request URL Responses High-level Data Description
GET /movies Return a list of ALL movies. JSON Data on list of ALL movies.
GET /movies/:title Return data about a single movie by title. Text Data indicating 'Successful GET request returning movie by title .
GET /movies/:year Return data by year. Text Data indicating 'Successful GET request returning movie by year .
GET /movies/:genre Return data by genre. Text Data indicating 'Successful GET request returning movie by genre' .
GET /director/:name Return data about a director by name. Text Data indicating 'Successful GET request of directors information.' .
POST /register Allow new users to register. Text Data indicating 'Successful POST to the server a new user registeration.' .
PUT /users/:id/:user_info Allow user to update the user info Text Data indicataing 'Successful PUT to the server a user information.' .
POST /users/:id/:favorites Allow user to add a movie to the list of favorites. Text Data indicataing 'Successful POST to the server a favorite movie on the user list.' .
DELETE /users/:id/:favorites All user to remove a movie from the list of favorites. Text Data indicataing 'Successful DELETE to the server a favorite movie on the user list.'.
DELETE /users/:id/unregister Allow existing user to deregister. Text Data indicataing 'Successful DELETE a user!' .
