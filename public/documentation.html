This is a Movie API App.

To build the server-side component of a “movies” web application. The web application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

Whenever users of mycommedyFlix are interacting with the application, the server-side of the application will be in use, processing their requests and performing operations against the data in the database. Users will be able to use the mycomedyFlix application whenever they like to read information about different movies or update their user information, for instance, their list of “Favorite Movies.”

HTTP Method URL Responses Request Body Format Response Body Format
GET /movies Return a list of ALL movies. None. JSON Data on list of ALL movies.
GET /movies/:title Return data about a single movie by title. None. JSON Data on movies by title.
GET /movies/genre/:name Return data by genre name. None. >JSON Data on info about a specific genre by name/title (e.g., “Thriller”).
GET /movies/director/:name Return data about a director by name. None. JSON Data on information about a specific director (bio, birth year, death year) by name.'
GET /users Return data about all users. None. JSON Data on all users.'
GET /users/:Username Return data about all users by username. None. JSON Data on all users by username.'
PUT /users/:Username Return data about Updated user's info by username. A JSON Object holding data about the users structured like: 
{
            id:1,
            Username:'john_smith',
            Email:'john@example.com',
            Password:'XXXXXX',
            Birthday: "1980-01-01"
},		 JSON Data on updated user by username.'
POST /users Return data about posted user. A JSON Object holding data about the user structured like: 
{
          id:1,
          Username:'john_smith',
          Email:'john@example.com',
          Password:'XXXXXX',
          Birthday: "1980-01-01"

    ]
